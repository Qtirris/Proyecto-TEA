Va a estar la rama MAIN que es pa que no vamos a tocar hasta que sepamos que todo sirve (En caso de que dañemos algo)

Esta la rama TEST que es en la que vamos a subir y combinar los codigos.

Los COMMIT "Guardan" los cambios en hechos justo a una descripción de lo que se hizo.

Los PUSH Suben los cambios a GitHUB ya que los commits son de manera local. Hacerlos primero a TEST.

OLA. ESTO ESTA EN TEST, POR LO QUE HASTA QUE NO HAYA UN MERGE NO SE VERA EN MAIN.

los PULL descargan los cambios al pc para trabajar. Se usa en caso de que alguien cambie algo, para tener tu la verion actualizada.

Una vez sepamos que todo sirve bien y que un push no haya dañado todo, hacemos un MERGE con MAIN. (Merge une los codigo y queda actializada la rama principal)

Usen la aplicación de escritorio, es lo más facil.
